# 知识库运行约定

维护以下通用闭环：

`输入 → 理解 → 知识 → 项目 → 成果 → 复盘 → 再输入`

## 目录职责

- `00_输入`：原始资料与结构化输入。不得改写 `原始资料` 中的文件。
- `01_理解`：对资料的摘要、证据拆分、解释和待确认问题。
- `02_知识`：可复用知识条目、索引和健康检查。
- `03_项目`：目标、任务、进度和相关资料。
- `04_成果`：报告、方案、产品、代码及其他完成结果。
- `05_复盘`：执行复盘、反馈、待确认事项和规则更新建议。
- `06_系统`：规则、自动化记录和检查点。
- `附件`：图片、文档和其他附件。
- `模板`：统一的中文文档模板。

## 执行要求

1. 保留来源、作者、链接、来源日期、收集日期、引用和原始内容。
2. 分开记录事实、解释、不确定内容和行动。
3. 使用 `06_系统/系统规则.md` 中的中文字段和状态。
4. 按实际关系链接输入、理解、知识、项目、成果和复盘。
5. 增量处理时只更新受影响的知识条目。
6. 记录自动变更、置信度和未解决问题。
7. 冲突、不确定合并、删除和大范围改写进入待确认队列。
8. 禁止在知识库中保存密钥。
9. 未经用户同意，不覆盖、移动或删除用户文件。
