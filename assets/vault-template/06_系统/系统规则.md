# 系统规则

## 命名与字段

- 文件名使用清晰中文标题，必要时以日期开头。
- 业务文档使用 `类型`、`状态`、`创建日期`、`更新日期`、`标签`。
- 日期使用 `YYYY-MM-DD`，日期时间使用 ISO 8601。
- 来源、事实、引用、解释和行动分开记录。

## 状态

- 输入：待处理、处理中、已整理、已归档。
- 理解：待整理、整理中、待确认、已完成。
- 知识：草稿、待验证、有效、需更新、已归档。
- 项目：构思、验证中、执行中、暂停、已完成、已归档。
- 成果：草稿、进行中、待验收、已完成、已归档。
- 复盘：待处理、处理中、待确认、已完成。

## 自动化边界

允许自动修复索引、链接、元数据、格式和摘要。冲突、合并、删除、无来源结论和大范围改写进入待确认队列。执行失败时不更新检查点。

## 隐私

API 密钥、Token、Cookie、密码和登录信息禁止写入 Markdown、附件或自动化报告。
