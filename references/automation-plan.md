# 自动化创建与执行

## 创建规则

使用可用的 Codex 自动化功能，定位根目录为当前仓库的本地项目。先检查已有自动化，名称相同时更新，避免重复创建。

创建两个使用本地时区的任务：

1. `知识库-每日信息收集`：每天 08:00。
2. `知识库-三天增量迭代`：每三天 20:30。

若仓库尚未注册为 Codex 项目，只请用户完成该操作。自动化功能不可用时，将状态记为待创建并写入 `06_系统/自动化任务.md`。

## 每日信息收集执行提示词

```text
使用当前仓库的 $obsidian-solo-os。运行 Skill 的 Horizon 导入程序，只把本次新生成的 Markdown 日报导入 00_输入/原始资料/资讯日报。GitHub 使用匿名额度，保持 X/Twitter 关闭，不修改 Horizon 源码，本任务不执行知识编译。在 06_系统/自动化/执行记录 中写入带日期的简短记录，包含执行时间、报告文件、可获取的来源数量和错误。禁止记录密钥或命令。
```

## 三天增量迭代执行提示词

```text
使用当前仓库的 $obsidian-solo-os。读取 06_系统/迭代状态.md，只处理上次成功检查点之后新增或变化的输入。执行 Skill 中 references/karpathy-optimization-prompt.md 的流程：整理新增输入，更新受影响的理解记录、知识条目、索引和项目关联，并进行知识健康检查。保留原始资料和用户文件。冲突、不确定合并、删除和无来源结论进入 05_复盘/待确认队列.md。

创建 06_系统/自动化/执行记录/YYYY/YYYY-MM-DD-HHmm-三天知识迭代.md，记录时间范围、处理来源、新建文件、更新文件、跳过文件、置信度、安全修复、待确认事项、下一步和新检查点。全部写入成功后才能更新 06_系统/迭代状态.md。
```

## 验收

- 两个任务均绑定当前仓库项目。
- 名称唯一，时间采用本地时区。
- 再次运行时更新已有任务。
- 执行记录使用带日期的文件名。
- 执行失败时不推进检查点。
