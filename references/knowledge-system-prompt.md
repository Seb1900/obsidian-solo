# 通用知识系统执行提示词

在受管模板安装后，于所选 Obsidian 仓库中执行以下提示词。

```text
将当前仓库搭建为通用个人知识系统。保留全部用户文件，只使用当前模板已安装的中文目录。

系统闭环：输入 → 理解 → 知识 → 项目 → 成果 → 复盘 → 再输入。

1. 输入
将文章、书籍、视频、逐字稿、资讯、灵感、对话、问题、反馈、项目经验、产品资料、研究资料和数据存入 00_输入。原始文件进入 00_输入/原始资料且保持不变。记录标题、来源、链接、作者、收集日期、资料类型、标签、状态、摘要和可利用方向。

2. 理解
在 01_理解 中整理输入。区分事实、引用、解释、不确定内容和行动建议；提取核心观点、证据、限制、关联问题和可复用价值。事实不充分时标记待确认。

3. 知识
将可复用内容增量编译到 02_知识/知识条目。相同概念更新原条目，边界明确的新概念才建立新条目。维护来源、证据日期、适用范围、反例、相关知识和修改记录，并同步索引与健康检查。

4. 项目
在 03_项目 中管理需要持续推进的事项。记录目标、阶段、任务、进度、截止日期、关联输入、关联知识、预期成果和复盘结果。项目类型由用户实际需求决定，不预设行业或业务流程。

5. 成果
在 04_成果 中保存研究报告、方案、文档、代码、产品、交付物及其他完成结果。每项成果链接来源、知识和项目，并记录版本、状态、使用场景和反馈。

6. 复盘
在 05_复盘 中记录执行结果、有效做法、问题、证据变化、待确认事项和后续行动。经过验证的结论回流到知识条目、方法和系统规则，保留历史结论及变更原因。

7. 运行节奏
持续收集输入，每日整理新增资料，每三天进行一次增量知识迭代，每周复盘项目与成果。每次自动执行都在 06_系统/自动化/执行记录 中记录处理范围、变更文件、置信度、待确认事项和下一步。

统一使用中文 Properties 字段和双向链接。禁止编造事实、删除用户文件、静默合并不确定概念、保存密钥或覆盖用户编写的内容。完成后更新索引。首次搭建不创建初始化报告、验收记录、兼容映射文档或空的执行记录；首次自动迭代发生时再创建对应执行记录。
```
