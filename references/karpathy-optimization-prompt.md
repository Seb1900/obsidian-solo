# 增量知识优化提示词

核心来源：`https://x.com/karpathy/status/2039805659525644595`。

核心方法是保留不可变原始资料，将新增内容增量编译为互相链接的 Markdown 知识条目、索引和派生成果，并定期进行健康检查。

```text
将当前 Obsidian 仓库作为增量编译的知识系统进行优化。

输入范围：
- 00_输入/原始资料 中保持不变的原始文件；
- 00_输入 中的结构化输入；
- 现有知识、项目、成果、复盘和用户反馈；
- 旧目录中用户已有的历史资料，只读取，不迁移、不改写。

对每个新增或发生变化的来源：
1. 保留原始文件，并为检查点记录内容标识。
2. 建立或更新理解记录，包含标题、作者、来源日期、收集日期、链接、摘要、证据类型和可靠性说明。
3. 提取独立概念，判断应更新现有知识条目还是建立新条目。
4. 只更新 02_知识/知识条目 中受影响的文件，记录定义、论点、证据、证据日期、限制、反例、相关知识、适用方向和更新时间。
5. 按需要建立输入、理解、知识、项目、成果、复盘和问题之间的双向链接。
6. 更新 02_知识/索引 中的来源、概念、最近变化、问题和项目索引。
7. 将有复用价值的报告、方案、方法和研究成果归档到对应知识条目或 04_成果。
8. 在本次执行记录和 06_系统/迭代状态.md 中记录变更文件、来源标识、置信度、待确认问题和后续建议。

检查孤立文档、失效链接、概念重叠、缺少来源或日期的论点、事实冲突、过时内容、缺少摘要、过长文档和潜在关联。

自动修复范围限于索引、双向链接、元数据、格式和简短摘要。事实冲突、不确定合并、无来源结论、删除和大范围改写进入 05_复盘/待确认队列.md。保留历史结论，并说明新证据带来的变化。

结束时记录：处理来源、新建和更新的知识条目、更新的索引、安全修复、待确认事项和下一步研究问题。
```
