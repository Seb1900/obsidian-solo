# 初始化时的一次性 Obsidian 操作

把以下信息合并到一条消息，并替换占位符。只提供文字步骤，不操作用户的 Obsidian。

- 仓库未创建：打开 Obsidian，创建或打开准备使用的仓库 `{SELECTED_VAULT}`。
- 社区插件确实未开启：打开“设置 -> 第三方插件”，关闭受限模式。检测到当前仓库已有有效的 `community-plugins.json` 时不要显示这一步。
- 必需插件只有 Claudian。仅在 Claudian 尚未安装或启用时显示该项。
- CLI 路径：从当前用户的环境变量、PATH、用户级包管理器目录和系统应用目录收集候选，逐个执行版本与 app-server 探测，只显示检测通过的 `{CODEX_CLI_PATH}`。禁止写死用户名或复用其他用户的路径。
- CLI 未找到：明确写“未找到可用的 Codex CLI，请先自行安装 Codex CLI 后重新初始化”，不要提供猜测路径。

消息结尾固定为：“完成缺少的插件安装后，可以回复‘初始化完成’或直接要求搭建知识库。不要把 API Key 发到聊天里。”

禁止调用 computer-use、浏览器控制或其他界面自动化。禁止创建 Claudian 对话、读取模型列表或发送测试请求。不要展示命令、依赖、脚本路径或内部检测信息。
